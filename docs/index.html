32-Channel EEG System

◈ System Overview

End-to-end architecture from electrodes to browser visualization.

⚡

Analog Front-End

4× TI ADS1299 providing 32 channels of 24-bit Sigma-Delta ADC with programmable gain (1–24×) and integrated lead-off detection.

⬡

Microcontroller

TM4C1294NCPDT ARM Cortex-M4F at 120 MHz. Interrupt-driven data acquisition with dual SPI buses, USB CDC, and UART debug.

⊕

Compute Module

CM5 (Raspberry Pi–class) running a C daemon for SPI transfer and frame parsing, piped to a Python/FastAPI real-time server.

⊞

Real-Time Streaming

IIR-filtered EEG data (4th-order Butterworth bandpass + 60 Hz notch) broadcast over WebSocket for browser-based visualization.

⚡ Operational Modes

Commands sent from CM5 to TM4C via SPI slave RX FIFO.

Command Mode Description Key Config
0x01 EEG Continuous 250 SPS, normal input multiplexer Gain 24, CHnSET = 0x68
0x02 STOP Halt any running acquisition mode —
0x03 Impedance One-shot lead-off sweep (6 µA, 31.2 Hz AC) Gain 1, LOFF = 0x0A
0x04 Test Internal short-circuit noise measurement Gain 24, CHnSET = 0x69

Mode Details

⚡

EEG Mode

CONFIG1: ADS1 = 0xF6 (master, CLK out, 250 SPS) / ADS2-4 = 0xD6 (slave)

CONFIG3: ADS1 = 0xEC (int ref + bias buffer) / ADS2-4 = 0xE8

BIAS: BIAS_SENSN = 0xFF on ADS1 only (all INxN → bias derivation)

📐

Impedance Mode

LOFF: 0x0A → 6 µA, 31.2 Hz AC excitation, 95% threshold

Sweep: 32 channels sequentially, 10 settling + 32 raw samples each

Output: Peak-to-peak packed into eeeData[112]

🔬

Test Shorted Mode

CHnSET: 0x69 → Gain=24, SRB2=closed, MUX=input shorted

Stream: Continuous RDATAC like EEG mode

Used for noise floor characterization.

▦ Data Frame Format

112-byte payload per DRDY sample, structured as 4 event blocks of 28 bytes each.

Offset Content Size Description
0–1[0x00][0x7E]2 BEvent 0 marker
2–3USB event ADS12 BUSB CDC event byte pair
4–27CH0 – CH724 B8 channels × 3 bytes (24-bit)
28–29[0x00][0x7D]2 BEvent 1 marker
30–31USB event ADS22 BUSB CDC event byte pair
32–55CH8 – CH1524 B8 channels × 3 bytes (24-bit)
56–57[0x00][0x7C]2 BEvent 2 marker
58–59USB event ADS32 BUSB CDC event byte pair
60–83CH16 – CH2324 B8 channels × 3 bytes (24-bit)
84–85[0x00][0x7B]2 BEvent 3 marker
86–87USB event ADS42 BUSB CDC event byte pair
88–111CH24 – CH3124 B8 channels × 3 bytes (24-bit)

Frame Structure Summary

Each event block: 2B marker + 2B USB + 24B data = 28 bytes

Total payload: 4 × 28 = 112 bytes/sample

SPI buffer is 135 bytes with up to 23 bytes of variable offset. The frame parser searches for the [0x00][0x7E] anchor and verifies the remaining 3 markers at EVENT_BLOCK intervals.

→ Data Path

Interrupt-driven flow from DRDY to WebSocket broadcast.

⚙ Firmware — TM4C1294

Bare-metal C firmware with interrupt-driven acquisition.

📄

main.c

Application entry point, RPIIntHandler (command dispatch), SysTickIntHandler, and super-loop for impedance sweep.

📄

ADS1299.c / .h

ADS1299 driver: ADSIntHandler (DRDY ISR), SSI0IntHandler, register commands (RREG/WREG/SDATAC/RDATAC), mode functions.

📄

Board.c / .h

GPIO, SPI (SSI2 master + SSI0 slave), UART debug, USB CDC, and SysTick initialization routines.

📄

Defines_ADS1299.h

Complete register address map and bit-field constants for the ADS1299 (CONFIG1-3, CHnSET, LOFF, BIAS, etc.).

⊞ Host Daemon — CM5

Linux C process that bridges the TM4C SPI data to Python via stdin/stdout.

📄

main.c (Raspi)

Multi-threaded SPI acquisition daemon. AcqThread polls DRDY, transfers 135 bytes, parses frames, writes .bin and CSV to stdout. SysMonitorThread reports CPU/RAM at 1 Hz.

📄

bacn_gpio.c / .h

GPIO control via sysfs: DRDY status polling (pin 5), START pulse generation (pin 4), initialization, and cleanup.

📄

sys_monitor.c / .h

Reads /proc/stat and /proc/meminfo to compute CPU and RAM usage percentages.

IPC Protocol (Python → C)

Binary stdin pipe 
0x01 → Start EEG acquisition
0x02 → Stop (any mode)
0x03 → Start impedance measurement
0x04 → Start internal test (ADS1299 test signal)

⊕ Python Server — FastAPI

Real-time EEG server with IIR filtering and WebSocket streaming.

🧮

Signal Processing

4th-order Butterworth bandpass (5–50 Hz) + 60 Hz notch (Q=30) cascaded as second-order sections (SOS). Per-channel IIR state is maintained across blocks for seamless filtering.

📡

WebSocket Streaming

Binary EEG data on /ws: 4-byte block number + float32 planar data (CH0×N, CH1×N, …, CH31×N). JSON status on /ws/status at 1 Hz.

⚙

Architecture

Reader thread: C stdout → hex parse → µV → block accumulation.
Filter task: async consumer — filters blocks, broadcasts binary.
Status task: async 1 Hz — broadcasts JSON metrics.

WebSocket Endpoints

WS /ws Binary EEG data stream (filtered, planar float32)
WS /ws/status JSON status stream at 1 Hz

WebSocket Binary Format (/ws)

Binary Layout 
[4-byte uint32 block_num LE]  +  [float32 planar EEG: CH0×N, CH1×N, ..., CH31×N]  +  [float32 planar USB: EVT0×N, ..., EVT3×N]

Block size N = 50 samples (200 ms at 250 SPS)
EEG data size = 32 × 50 × 4 = 6400 bytes
USB data size = 4 × 50 × 4 = 800 bytes
Total message = 4 + 6400 + 800 = 7204 bytes

WebSocket JSON Format (/ws/status)

JSON 
{
  "cpu":      12.3,

▶ Quick Start

Get MonEEG running from source.

1. Firmware (TM4C1294)

Open MonEEE/ as a project in Code Composer Studio (TI CCS). Build and flash to the TM4C1294NCPDT board.

2. Host Daemon (CM5 / Raspberry Pi)

Bash 
# Compile the C daemon
cd Raspi
gcc -o moneee main.c Drivers/bacn_gpio.c Drivers/sys_monitor.c -lpthread

3. Python Server

Bash 
# Install dependencies
pip install fastapi uvicorn numpy scipy jinja2

# Run the server
python main.py

4. Access the Dashboard

Open a browser and navigate to http://<CM5-IP>:8000. The dashboard will connect via WebSocket and display real-time filtered EEG waveforms.
